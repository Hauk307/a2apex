🪪 Agent Card Validation

The Agent Card is the foundation of the A2A protocol — it's how agents discover and understand each other. A2Apex validates your Agent Card against the complete A2A specification (v0.3) and provides a compliance score.

What Gets Validated?

A2Apex performs comprehensive validation across all Agent Card fields. Issues are categorized into three severity levels:

Severity Description Impact on Score
Error Must fix — breaks protocol compliance -10 points each
Warning Should fix — may cause interoperability issues -2 points each
Info Suggestion for best practices No penalty

Required Fields

These fields must be present in your Agent Card. Missing any of these will result in errors:

Field Type Description
name string Human-readable name of the agent
url string (URL) Base URL of the A2A endpoint
version string (semver) Agent version (e.g., "1.0.0")
capabilities object Agent capability flags
skills array List of agent skills (at least one required)

Capabilities Object

The capabilities object describes what your agent supports:

{
  "capabilities": {
    "streaming": true,
    "pushNotifications": false,
    "multiTurn": true,
    "stateTransitionHistory": false
  }
}

Skills Array

Each skill must have at minimum id and name:

{
  "skills": [
    {
      "id": "research",
      "name": "Web Research",
      "description": "Search the web and synthesize findings",
      "tags": ["search", "research", "web"]
    }
  ]
}

Recommended Fields

These fields are not required but are strongly recommended. Missing them will generate warnings:

Field Type Description
description string Human-readable description of the agent
protocolVersion string A2A protocol version (e.g., "0.3")
defaultInputModes array MIME types the agent accepts
defaultOutputModes array MIME types the agent produces
provider object Organization providing the agent
documentationUrl string (URL) Link to agent documentation

Optional Fields

These fields are optional but provide additional functionality:

Field Type Description
iconUrl string (URL) URL to agent icon/logo
preferredTransport string JSONRPC, GRPC, or HTTP+JSON
securitySchemes object Authentication methods (OpenAPI 3.0 format)
security array Required security schemes
supportsAuthenticatedExtendedCard boolean Whether auth reveals more info

Security Scheme Validation

A2Apex validates security schemes against the OpenAPI 3.0 specification. Supported types:

API Key

{
  "securitySchemes": {
    "api_key": {
      "type": "apiKey",
      "name": "X-API-Key",
      "in": "header"
    }
  }
}

HTTP Bearer (JWT)

{
  "securitySchemes": {
    "bearer_auth": {
      "type": "http",
      "scheme": "bearer",
      "bearerFormat": "JWT"
    }
  }
}

OAuth2

{
  "securitySchemes": {
    "oauth2": {
      "type": "oauth2",
      "flows": {
        "authorizationCode": {
          "authorizationUrl": "https://auth.example.com/authorize",
          "tokenUrl": "https://auth.example.com/token",
          "scopes": {
            "read": "Read access",
            "write": "Write access"
          }
        }
      }
    }
  }
}

How Scoring Works

The compliance score is calculated as follows:

  • Base score: 100 points
  • Each error: -10 points
  • Each warning: -2 points
  • Info items: No penalty
  • Minimum score: 0

A score of 80+ is considered good compliance. 100 means perfect compliance with no errors or warnings.

💡 Score Interpretation

90-100: Excellent — Ready for production
80-89: Good — Minor improvements recommended
60-79: Fair — Fix warnings before deployment
Below 60: Poor — Fix errors before use

Common Issues & Fixes

❌ Missing required field: skills

Your Agent Card must include at least one skill.

{
  "skills": [
    {
      "id": "default",
      "name": "Default Skill",
      "description": "Primary agent capability"
    }
  ]
}

❌ Invalid URL format

URLs must be valid HTTP/HTTPS URLs. Localhost is allowed for development.

{
  "url": "https://api.example.com/a2a"
}

❌ Invalid semver version

Version must follow semantic versioning format (X.Y.Z).

{
  "version": "1.0.0"
}

⚠️ Missing description

Add a clear description so other agents understand what yours does.

{
  "description": "An AI assistant that helps with research and analysis tasks."
}

⚠️ Missing protocolVersion

Specify which A2A protocol version your agent implements.

{
  "protocolVersion": "0.3"
}

⚠️ HTTP URL (should use HTTPS)

Production agents should use HTTPS for security.

Complete Agent Card Example

Here's a fully compliant Agent Card that scores 100/100:

{
  "name": "Research Assistant",
  "description": "An AI agent that searches the web, analyzes documents, and synthesizes research findings.",
  "url": "https://api.example.com/a2a",
  "version": "1.2.0",
  "protocolVersion": "0.3",
  "capabilities": {
    "streaming": true,
    "pushNotifications": false,
    "multiTurn": true,
    "stateTransitionHistory": true
  },
  "defaultInputModes": ["text/plain", "application/json"],
  "defaultOutputModes": ["text/plain", "application/json", "text/markdown"],
  "skills": [
    {
      "id": "web-research",
      "name": "Web Research",
      "description": "Search the web and synthesize findings on any topic",
      "tags": ["search", "research", "web", "synthesis"]
    },
    {
      "id": "document-analysis",
      "name": "Document Analysis",
      "description": "Analyze PDFs, documents, and extract key information",
      "tags": ["documents", "analysis", "extraction"]
    }
  ],
  "provider": {
    "organization": "Example Corp",
    "url": "https://example.com",
    "contactEmail": "ai@example.com"
  },
  "documentationUrl": "https://docs.example.com/research-agent",
  "iconUrl": "https://example.com/agent-icon.png",
  "securitySchemes": {
    "api_key": {
      "type": "apiKey",
      "name": "X-API-Key",
      "in": "header"
    }
  },
  "security": [
    {"api_key": []}
  ]
}

Validating via API

You can validate Agent Cards using the REST API:

Validate from URL

curl -X POST https://api.a2apex.io/api/validate/agent-card \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-agent.example.com"}'

Validate JSON directly

curl -X POST https://api.a2apex.io/api/validate/agent-card \
  -H "Content-Type: application/json" \
  -d '{
    "json": {
      "name": "My Agent",
      "url": "https://api.example.com/a2a",
      "version": "1.0.0",
      "capabilities": {},
      "skills": [{"id": "default", "name": "Default"}]
    }
  }'

See the API Reference for full details on the validation endpoint.
