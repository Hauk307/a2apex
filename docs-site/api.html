📡 API Reference

The A2Apex REST API provides endpoints for validating Agent Cards, running compliance tests, and managing authentication. All endpoints accept JSON and return JSON responses.

Base URL

https://api.a2apex.io

Authentication

Most endpoints require authentication via API key or JWT token:

API Key (Header)

curl -H "X-API-Key: your_api_key" https://api.a2apex.io/api/validate/agent-card

API Key (Query Parameter)

curl "https://api.a2apex.io/api/validate/agent-card?api_key=your_api_key"

JWT Bearer Token

curl -H "Authorization: Bearer your_jwt_token" https://api.a2apex.io/api/validate/agent-card

Rate Limits

Tier Limit Window
Free (no API key)10 requestsper minute
Pro (API key)100 requestsper minute
EnterpriseCustomper minute

Rate limit headers are included in all responses:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1709064000

Validation Endpoints

POST /api/validate/agent-card

Validate an Agent Card against the A2A specification.

Request Body

FieldTypeDescription
urlstringURL to fetch the Agent Card from
jsonobjectAgent Card JSON to validate directly

Provide either url or json, not both.

Example Request

curl -X POST https://api.a2apex.io/api/validate/agent-card \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your_api_key" \
  -d '{"url": "https://agent.example.com"}'

Example Response

{
  "is_valid": true,
  "score": 92.5,
  "summary": {
    "errors": 0,
    "warnings": 3,
    "info": 2
  },
  "errors": [],
  "warnings": [
    {
      "field": "protocolVersion",
      "message": "Missing recommended field",
      "suggestion": "Add protocolVersion: \"0.3\"",
      "spec_reference": "Section 6.1"
    }
  ],
  "info": [...],
  "agent_card": { ... }
}
POST /api/validate/state-transition

Validate A2A task state transitions.

Request Body

// Validate a sequence of states
{"states": ["submitted", "working", "completed"]}

// Or validate a single transition
{"from_state": "submitted", "to_state": "working"}

Testing Endpoints

POST /api/live-test

Run live endpoint tests against an A2A agent.

Request Body

FieldTypeRequiredDescription
agent_urlstringYesBase URL of the A2A agent
auth_headerstringNoAuthorization header value
timeoutfloatNoRequest timeout in seconds (default: 30)

Example

curl -X POST https://api.a2apex.io/api/live-test \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your_api_key" \
  -d '{
    "agent_url": "https://agent.example.com",
    "auth_header": "Bearer agent_token",
    "timeout": 30
  }'
POST /api/test/auth

Run authentication and security tests.

Tests security scheme declarations, API key auth, HTTP bearer, OAuth2 flows, unauthenticated request handling, and HTTPS enforcement.

POST /api/test/errors

Run error handling tests.

Tests JSON-RPC 2.0 error compliance: malformed JSON, invalid requests, unknown methods, invalid parameters, non-existent tasks, and oversized payloads.

POST /api/test/streaming

Run SSE streaming tests.

Tests SSE connection, event format validation, state transition ordering, artifact streaming, stream termination, and timeout handling.

POST /api/test/performance

Run performance tests.

Tests Agent Card fetch latency, message send latency, concurrent request handling, task isolation, and cold/warm start performance.

POST /api/test/full

Run the FULL compliance test suite (all tests).

Test Reports

GET /api/test-report/{report_id}

Get a previously generated test report by ID.

GET /api/test-reports

List recent test reports.

Query Parameters

ParameterTypeDefaultDescription
limitinteger20Maximum reports to return

Chat / Debug Endpoint

POST /api/chat

Send a chat message to an A2A agent and return structured results.

Request Body

{
  "agent_url": "https://agent.example.com",
  "message": "Hello, how can you help me?",
  "context_id": "optional-context-id",
  "task_id": "optional-task-id",
  "auth_header": "Bearer token",
  "timeout": 30
}

Response

{
  "success": true,
  "raw_request": { ... },
  "raw_response": { ... },
  "duration_ms": 245.3,
  "http_status": 200,
  "parsed": {
    "task_id": "task-123",
    "context_id": "ctx-456",
    "status": "completed",
    "agent_message": "I can help you with...",
    "artifacts": [...]
  }
}

Authentication Endpoints

POST /api/auth/register

Register a new user account.

{
  "email": "user@example.com",
  "password": "minimum8chars",
  "name": "Your Name"
}
POST /api/auth/login

Login with email and password.

{
  "email": "user@example.com",
  "password": "yourpassword"
}

Response

{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "name": "Your Name",
    "plan": "free",
    "api_key": "a2apex_...",
    "created_at": "2026-02-27T12:00:00Z"
  }
}
GET /api/auth/me

Get the current authenticated user's profile.

POST /api/auth/refresh

Refresh the JWT access token.

POST /api/auth/regenerate-api-key

Generate a new API key (invalidates the old one).

Utility Endpoints

GET /api/health

Health check endpoint (no auth required).

{
  "status": "healthy",
  "service": "a2apex",
  "version": "0.2.0",
  "timestamp": "2026-02-27T12:00:00Z"
}
GET /api/state-machine/diagram

Get the A2A state machine diagram and valid transitions.

GET /api/state-machine/next/{current_state}

Get valid next states from a given state.

Error Responses

Error responses follow a consistent format:

{
  "error": "Error Type",
  "message": "Detailed error message",
  "detail": "Additional context (optional)"
}

HTTP Status Codes

CodeMeaning
200Success
201Created (registration)
400Bad Request (invalid input)
401Unauthorized (invalid/missing auth)
404Not Found
429Too Many Requests (rate limited)
500Internal Server Error
