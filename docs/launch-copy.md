## 1. Hacker News — "Show HN"

**Title:** Show HN: A2Apex – The trust layer for AI agents (test, certify, get listed)

**Body:**

Hey HN,

I built A2Apex (https://a2apex.io) — a testing and reputation platform for AI agents built on Google's A2A protocol.

**The problem:** AI agents are everywhere, but there's no standard way to verify they work. You can't browse trusted agents. You can't prove your agent is compliant. There's no "SSL certificate" moment for the agent ecosystem.

**What A2Apex does:**

- **Test** — Point it at any A2A agent URL. It runs 50+ automated compliance tests (agent card validation, live endpoint testing, state machine verification, streaming, auth, error handling).

- **Certify** — Get a 0-100 trust score with Gold/Silver/Bronze badges you can embed in your README or docs.

- **Get Listed** — Every tested agent gets a public profile page with trust scores, skills, test history, and embeddable badges. Developers browse the Agent Directory to find agents they can trust.

Think of it as the intersection of SSL Labs (testing), npm (directory), and LinkedIn (profiles) — but for AI agents.

**Stack:** Python/FastAPI backend, vanilla JS frontend, SQLite. No frameworks, no build tools. The whole thing runs on a Mac mini in Wyoming.

Free tier: 5 tests/month. Pro: $29/mo unlimited. Try it: https://a2apex.io

Would love feedback from anyone building A2A agents or thinking about agent interoperability.
